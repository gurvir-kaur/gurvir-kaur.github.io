INTERNET CONNECTION and WEB SOCKET

Internet connection is just point to point connection of two points on the Internet, full duplex connection data flows in both the directions after the connection has been established. It is reliable.
Socket, it's just an end, end to the point to point connection, so for an internet connection we need two sockets, one on the client side and the other on the server,
Breaking down the sockets in two parts, they are IP address and port number for example client socket can be 123.76.87.8:9898, so on the left of colon is the IP address of the client and on the right side is a port number.

What are port numbers?
For client side these can be randomly assigned by the OS (The Ephemeral Port), but for server side for example youtube.com:80, port number 80 is standard port number for all of the web traffic.







So why does client have random port number but the server side has 80?
Because the server side has to be more reliable and available for all the time.

How do web browsers work?

  1. User Interface- how it looks.
  2. Browser engine-facilitates comm. b/w UI and the rendering engine.
  3. Rendering engine-takes all that html and CSS and puts something on the screen.
  4. Networking blocks (http) - requests stuff all over the internet and also asks DNS to resolve domain names.
  5. JS (java script) Interpreter- if a web app is developed there are a lot of scripts running, so an interpreter is needed to interpret and execute the JS.
  6. UI Backend- real drawing of lot of things, if we say we want to draw rectangle, a rectangle is drawn on monitor.
  7. Data- browser save info, like remember me, search history, set up data, persistence layer.

Five steps in rendering engine :

  1. Resource gathering- webpages, using networking model, html, pics to get what we need to display.
  2. Parse HTML and create DOM TREE- document object model, its official thing regulated by W3, so in this step we take all that data from step one and create a tree of DOM objects.
  3. Create render tree from DOM tree- it hides the things which are not to be seen from html code so a DOM tree is converted to the render tree of what actually needs to be seen. Puts all the stylings of CSS and html to the effect.
  4. Layout - position the rendered tree and give them size.
  5. Painting - taking all the info and sending to UI backend to actually draw on the monitor.

Core UI gets to the OS to use the draw function. Handle z axis, over and under DOM Tree constantly changes and needs to be re-parsed dynamically.

HTTP

(Hyper Text Transfer Protocol)

  1. Follows client/server model or request/response model, so the browser we use is the client to the protocol and the web applications like YouTube or Facebook are servers, that are sitting on the cloud on the internet.
  2. It is stateless protocol. Means every transaction is independent of the other. Even though http is stateless we get the functionality of the states through other things such as cookies or sessions.
  3. It is an application layer, sits above transportation layer, it uses TCP as its transport level protocol. TCP defines how data is transmitted.
  4. Client action, whenever client request something from server always specify action, like GET (read only, get a page or resource) /POST (creating something new) /DELETE.
  5. Server status code, define response static codes that the server makes to client. 200 ok, 404 not found.
  6. Headers, tiny bits of custom info that are sent along with the request and with responses to pull the operation of request, for example one header that can be included on the request header side is content type. Response header can be cache control, optimization technique of internet and determines how different resources are cached by all the different clients. There is no rulebook for headers, these can be custom to a particular website.

Session & Cookies

Cookies are something which is a function of state and that is a part of client side (browser) and session is related to the web server, web client(browser): server ∷ we: Starbucks ∷ cookies: session.

Every single http request sent by our browser carry a cookie with it so that the server can do something smart with it. Http is originally a stateless protocol but sessions and cookies make it state full.
Cookies are always client side but session is mostly server side but sometimes sessions are also stored on the client side, that makes it real fast but there is lack OS space.

















Security

Asset can be anything like hardware, software or data. The perceived value of an asset depends upon the ease with which the asset can be replaced.
One of the major goal of information security is to reduce the security risks but along with it also to protect the valuable assets.
To study methods of assets protection, we use a vulnerability-threat-control framework.
Vulnerability is a weakness in the system, and can be exploited to cause harm or loss.
Threat, a set of circumstances that has the potential to cause harm or loss. Control, an action, device, or procedure that eliminates or reduce a vulnerability. Threats are blocked by controlling vulnerability!
CIA= the security triad, so the threats can be thought to interfering with CIA of an information system.
Confidentiality - the ability of a system to ensure that assets are viewable only by authorized parties.
Integrity - the ability of a system to ensure that assets are modified only by the authorized parties.
Availability - the ability of a system to ensure that assets are only usable and assessable by authorized parties.
Aside from CIA other system properties include :
Authentication - the ability of a system to confirm the identity of a sender.
Nonrepudiation - property of a system in which the sender can't convincingly deny having sent a message.
Auditability - ability of a system to trace all actions related to a given asset.

Harmful Acts

harm to a system can be caused by 4 different ways-

  1. Interception - intercept valuable information flowing over a network.
  2. Interruption- disrupt the information systems ability to carry out its tasks.
  3. Modification - modify system or modify its information assets without being properly authorized to do so.
  4. Fabrication - fabricate new identity or asset to cause harm to the system as a whole.

Each of the above acts are harmful acts and can affect the system's ability to ensure CIA.

For an attack to be successful attackers need MOM!

  1. Method- skill knowledge tool with which to attempt an attack.
  2. Opportunity - time and access to attempt an attack.
  3. Motive - a reason to attempt an attack.

If any one of three is eliminated, attack will not be successful

Method of Defence

Six approaches to defend our system-

  1. Prevent attack- block attack/close vulnerability.
  2. Deter attack- make attack harder if not fully prevented.
  3. Deflect attack- make another target more attractive than this target.
  4. Mitigate attack- make the impact of attack less severe.
  5. Detect attack- during or after (also important, we can learn and recover from attack).
  6. Recover from attack- backup copies of data or organizational protocols that allow us to quickly recover from successful attack.
